# Data Engineering Training Journey

This repository reflects my learning progress during the data engineering training, where I moved step by step from basic concepts to working on complete data workflows using PySpark and Databricks.

---

# WEEK 0

---

## Phase 0

Completed the Databricks Associate Data Engineering course.  
This phase helped me get familiar with the platform and understand how to run basic operations.

---

## Phase 1

📚 Topics covered: SELECT, FILTER, GROUP BY  

✅ Key learnings:  
I learned how to retrieve and filter data properly.  
Also got an idea of how grouping helps in summarizing data.  

⚠️ Challenges faced:  
Understanding how grouped results are formed was not very clear initially.  
Also needed some time to adjust to PySpark syntax.

---

## Phase 2

📚 Topics Covered:  
Joins and aggregations in PySpark, along with converting SQL logic into DataFrame operations.  

✅ Key Learnings:  
Learned how to combine data from different tables and perform calculations on it.  
Also improved confidence in writing PySpark instead of SQL.  

⚠️ Challenges:  
At times, join results were not as expected, so I had to recheck conditions.  
Understanding how joins affect data required some practice.

---

## Phase 3

📚 Topics Covered:  
ETL basics, reading data from files, cleaning data, validation, joins, and aggregations.  

✅ Key Learnings:  
Learned how to handle raw data and make it usable.  
Understood why cleaning and validation are important before analysis.  

⚠️ Challenges:  
Faced issues with incorrect data formats.  
Also had to carefully apply transformations in the correct order.

---

## Phase 4

📚 Topics Covered:  
End-to-end pipeline creation, data cleaning, joins, aggregations, segmentation, and reporting.  
4A: different bucketing techniques  

✅ Key Learnings:  
Understood how different steps come together to form a complete pipeline.  
Learned how to apply simple business rules for segmentation.  

⚠️ Challenges:  
Sometimes results were not accurate due to missing or incorrect data.  
Had to revisit steps to fix and improve the final output.

---

## Phase 5

📚 Topics Covered:  
Databricks setup, working with a real dataset, combining tables, performing analysis, and generating reports.  

✅ Key Learnings:  
Learned how to handle a real dataset from start to end.  
Got a better idea of how different tables are connected.  
Also used window functions for more detailed analysis.  

⚠️ Challenges:  
Understanding dataset structure took time.  
Some required data was not directly available, so I had to explore more.  
Also needed practice to correctly apply window functions.
